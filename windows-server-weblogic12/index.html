Weblogic 安装配置

安装版本：Weblogic 12.1.3

安装环境：Windows Windows Server2012R2

安装目录：C:\Oracle\Middleware\Oracle_Home

http://ip:7001/console

账户/密码：weblogic/weblogic123

关于Weblogic

Oracle WebLogic应用服务器产品系列是业界最全面的开发、部署和集成企业管理软件的平台。该产品系列的核心是Oracle WebLogic服务器，它是一个功能强大和可扩展的Java EE服务器。它与Oracle应用服务器以及诸如Oracle JRockit和Oracle Coherence这样的其他性能增强产品结合在一起构成了Oracle WebLogic套件。此外，Oracle WebLogic应用网格为极限事务处理 (XTP) 提供了必要的Java基础架构。

Oracle WebLogic套件和Oracle WebLogic应用网格是Oracle融合中间件的战略性应用服务器产品。 在Oracle中间件战略中可了解有关这些产品及其作用的更多信息。Oracle应用服务器的客户可从持续的开发和Oracle所做的随着时间的推移将其最佳功能与Oracle WebLogic服务器相集成的承诺中受益。

Weblogic的版本不同，会导致Oracle数据库的驱动不同，驱动不同可能会引起一些问题。

关于卸载Weblogic

开始 –> Oracle –> Oracle Home –> Uninstall Oracle Middleware，便开始下载了。

建议，不要删除原来的Weblogic的安装目录，因为重新安装之后，仍需要部署项目，这时，原来是domain里面setDomainEnv.cmd等文件中的配置可以直接拷贝出来。

关于安装Weblogic

下载地址：http://www.oracle.com/technetwork/middleware/weblogic/downloads/wls-for-dev-1703574.html

下载Generic版本：fmw_12.1.3.0.0_wls.jar

以前的安装介质中有包含jdk的，比如exe，bin方式的介质，新版如12c版本的所有介质是.jar的形式，不再包括jdk。所以在安装Weblogic之前一定要先安装好jdk，设置好JAVA_HOME，PATH等相关环境变量。

关于JDK的安装，最好不要使用默认的安装目录，一般会安装在Program Files目录下，该目录会导致JDK的路径中出现空格，在使用Weblogic配置一下Scheduler的时候需要给出JDK路径，但要求JDK路径不能有空格，有一些方式可以解决，但最好还是重新指定一个目录吧！

安装方式：java -jar fmw_12.1.3.0.0_wls.jar，就可以执行安装。

注意：需要使用Oracle官方文档指定支持的JDK。若安装不成功需要检查环境变量。
